Tema 1: Elementos de paginas web

I. Origenes de la web

La idea de la web surgió en 1989 cuando el científico británico Tim Berners-Lee propuso un sistema de gestión de información basado en hipertexto mientras trabajaba en el CERN, un laboratorio de física en Suiza

1. web 2.0

surgió en la década de 2000 para describir la transición de la web estática de la primera generación a una web más dinámica y centrada en la colaboración y la interacción del usuario.

2. web 3.0

fue acuñado por Gavin Wood, cofundador de Ethereum, poco después del lanzamiento de Ethereum en 2014. La idea surgió como una solución a la falta de confianza en la web centralizada, donde unas pocas grandes empresas controlan gran parte de la información y los datos.

3. La W3C y los estandares

es una organización internacional que desarrolla estándares para garantizar el crecimiento a largo plazo de la Web. Fundada por Tim Berners-Lee, el inventor de la Web, W3C reúne a diversas empresas, universidades y grupos de interés para crear directrices técnicas que aseguren la interoperabilidad y el acceso universal de la Web. Entre los estándares más conocidos y ampliamente utilizados que ha desarrollado se encuentran HTML, CSS y XML.

II. Lenguajes de etiquetas

El lenguaje de etiquetas se refiere comúnmente a HTML (HyperText Markup Language), que es el estándar para crear páginas web. Con HTML, puedes estructurar tu contenido mediante etiquetas, que son básicamente comandos que indican al navegador cómo mostrar texto, imágenes y otros elementos en una página web.

1. SGML

es un estándar ISO para definir lenguajes de marcado. Fue desarrollado en los años 80 y permite a los usuarios crear y gestionar documentos con una estructura definida.

2. XML

es un lenguaje de marcado que define reglas para la codificación de documentos en un formato que es tanto legible por humanos como por máquinas.

3. HTML

es el lenguaje estándar utilizado para crear páginas web. Permite estructurar y presentar contenido en la web mediante etiquetas y atributos. Estas etiquetas organizan el contenido en elementos como párrafos, encabezados, enlaces, imágenes y mucho más.

4. XHTML

es una versión más estricta y bien estructurada del HTML (HyperText Markup Language), que es el lenguaje utilizado para crear y diseñar páginas web.

5. HTML5

es un Lenguaje de Marcado de Hipertexto (HyperText Markup Language) que funciona con las bases del HTML de siempre, pero que cuenta con etiquetas adicionales que permiten crear sitios webs más modernos y más compatibles con las tecnologías digitales actuales.

III. Otros lenguajes relacionados al desarrollo web

1. CCS/CSS3

es la última evolución del lenguaje de hojas de estilo utilizado para definir el aspecto y el formato de un documento escrito en un lenguaje de marcado como HTML.

2. javaScript

es un lenguaje de programación ampliamente utilizado en el desarrollo web. Es un lenguaje de scripting que permite a los desarrolladores crear contenido dinámico y interactivo en páginas web, como animaciones, formularios, juegos y aplicaciones web.

3. Ajax

es una técnica utilizada en el desarrollo web para crear aplicaciones más interactivas y dinámicas. Básicamente, permite actualizar partes de una página web sin tener que recargar toda la página

4. Tecnologias y lenguajes del lado del servidor

son aquellos utilizados para manejar la lógica, el almacenamiento de datos y las operaciones en el servidor, en lugar del lado del cliente (navegador web).

5. Navegadores

es una aplicación que te permite acceder y visualizar sitios web en internet. Los navegadores interpretan el código de las páginas web y lo presentan en un formato visual amigable.

6. Motores renderizados

es un componente crucial en navegadores web y software gráfico. Su función principal es transformar el código (HTML, CSS, JavaScript) en imágenes y texto que puedan visualizarse en pantalla.

IV. herramientas de desarrollo

son programas y aplicaciones que los desarrolladores de software utilizan para crear, probar, depurar y mantener aplicaciones y sistemas.

1. Software WYSIWYG

Se refiere a un tipo de software que permite a los usuarios crear y editar documentos o interfaces de usuario de manera visual.

2.Otras alternativas

i.Figma

es una herramienta de diseño colaborativo basada en la nube que permite a los usuarios crear interfaces de usuario, prototipos interactivos y colaborar en proyectos de diseño de forma simultánea desde cualquier lugar.

ii. InVision

es una plataforma de diseño digital muy popular entre diseñadores y equipos creativos. Se utiliza principalmente para crear prototipos interactivos, colaborar en proyectos de diseño y gestionar flujos de trabajo de diseño.

iii. Zeplin

es una herramienta de colaboración diseñada específicamente para equipos de diseño y desarrollo. Facilita la entrega de diseños, permitiendo a los diseñadores y desarrolladores trabajar juntos de manera más eficiente y efectiva.

iv. Github

es una plataforma de desarrollo colaborativo para alojar y revisar código, gestionar proyectos y construir software. Utiliza el sistema de control de versiones Git, lo que permite a los desarrolladores trabajar en un proyecto de manera conjunta y a la vez poder gestionar cambios y versiones de manera eficiente.

3. Inspeccion de codigo desde el navegador

se refiere a la capacidad de examinar y modificar el código fuente de una página web directamente en tu navegador. Esta función es especialmente útil para desarrolladores web, diseñadores y cualquiera interesado en entender cómo está construida una página web.

Tema 2: el diseño de una pagina web

es un conjunto de documentos digitales interconectados a través de enlaces, que se encuentran en la red mundial de internet. Estos documentos pueden incluir texto, imágenes, videos y otros elementos multimedia, y están diseñados para ser visualizados en un navegador web.

I. tipos de paginas web

Páginas Web Personales: Son creadas por individuos para compartir información personal, intereses, hobbies, o un portafolio profesional. Blogs: Sitios donde los autores publican contenido regularmente sobre temas específicos. Pueden ser personales, profesionales o temáticos. Sitios Corporativos: Diseñados para proporcionar información sobre una empresa, sus productos o servicios, su misión y su equipo. Tiendas en Línea (E-commerce): Páginas dedicadas a la venta de productos o servicios a través de internet, como Amazon o eBay. Sitios de Noticias: Proveen las últimas noticias y artículos sobre diferentes tópicos, desde política hasta entretenimiento. Redes Sociales: Plataformas donde las personas pueden conectarse, compartir contenido y comunicarse, como Facebook, Instagram o Twitter. Portafolios: Muestran el trabajo de profesionales, como fotógrafos, diseñadores gráficos, artistas, etc. Foros y Comunidades: Lugares donde los usuarios pueden interactuar y discutir sobre temas específicos. Ejemplos incluyen Reddit y Quora. Educativos: Ofrecen cursos, recursos educativos y materiales de aprendizaje, como Coursera o Khan Academy. Entretenimiento: Incluyen contenido de entretenimiento, como videos, música, juegos en línea, etc. Ejemplo: YouTube, Netflix. Landing Pages: Páginas específicas creadas para campañas de marketing que capturan la información del visitante a través de formularios. Wikis: Páginas colaborativas donde los usuarios pueden editar y agregar contenido, como Wikipedia.

II. puntos basicos en la elaboracion de una pagina web

Planificación: Define el propósito de la página web y el público objetivo. Esto ayudará a guiar el diseño y el contenido. Estructura: Esquema de la estructura de tu sitio web, incluyendo la navegación principal y las páginas clave (inicio, sobre nosotros, servicios, contacto, etc.). Diseño: Selecciona una paleta de colores, fuentes y estilos visuales que representen la identidad de tu marca. Herramientas como Figma o Adobe XD pueden ser útiles. Contenido: Redacta textos, crea imágenes y otros medios. Asegúrate de que el contenido sea relevante y atractivo para tus visitantes. Desarrollo: Es hora de construir el sitio. Puedes usar un creador de sitios web como WordPress, Wix, o aprender HTML, CSS y JavaScript para un enfoque más personalizado. SEO (Optimización de Motores de Búsqueda): Optimiza tu contenido para que los motores de búsqueda puedan encontrar y clasificar tu sitio. Esto incluye el uso de palabras clave, etiquetas meta y un contenido de calidad. Pruebas: Asegúrate de que tu sitio web funcione correctamente en diferentes navegadores y dispositivos. Revisa los enlaces, la navegación y la carga de las páginas. Lanzamiento: Publica tu sitio web y asegúrate de anunciarlo a través de redes sociales y otros canales para atraer visitantes. Mantenimiento: Actualiza regularmente tu sitio con contenido nuevo y realiza ajustes según sea necesario para mantenerlo funcionando correctamente y relevante.

II.Elementos de estructura para el diseño de una página Web: Concepto

1.Regla utilizada para validar y certificar una página web

Accesibilidad Web: La W3C (World Wide Web Consortium) tiene las Pautas de Accesibilidad para el Contenido Web (WCAG) que se utilizan para asegurar que las páginas web sean accesibles para personas con discapacidades. Seguridad Web: Certificaciones SSL/TLS para asegurar la transmisión de datos. OWASP (Open Web Application Security Project) proporciona pautas para asegurar aplicaciones web contra vulnerabilidades comunes. SEO (Optimización para Motores de Búsqueda): Google y otros motores de búsqueda tienen directrices específicas que ayudan a asegurar que un sitio web esté optimizado para la búsqueda y la visibilidad en línea. Cumplimiento Legal: Dependiendo de la región, pueden existir normativas legales, como GDPR en Europa para la protección de datos personales. Validación de HTML y CSS: Usar las herramientas de validación de W3C para asegurarse de que el código del sitio cumple con los estándares web. Rendimiento Web: Herramientas como Google PageSpeed Insights o GTmetrix pueden ayudar a medir y mejorar el rendimiento y la velocidad del sitio web.

2.Reglas para escribir HTML.

Utiliza las etiquetas correctas y respeta el formato semántico de la web. Mantén la estructura lo más sencilla posible usando la menor cantidad de etiquetas. Evita usar comentarios.
